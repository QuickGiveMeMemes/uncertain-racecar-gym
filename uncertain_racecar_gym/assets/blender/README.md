# Blender Replay Notes

Use the replay bundle generated by `uncertain-racecar-export-replay` to drive an offline scene in Blender.

Recommended pipeline:
1. Export a replay bundle with `uncertain-racecar-export-replay`.
2. Render it headlessly with `uncertain-racecar-render-blender`.
3. Review the generated `.mp4` and optional `.blend` scene.

By default the Blender renderer uses a bundled online-downloaded car mesh:

- `uncertain_racecar_gym/assets/blender/vendor/kenney_raceCarRed.glb`
- License text: `uncertain_racecar_gym/assets/blender/vendor/LICENSE_Kenney_Racing_Kit.txt`

You can override that with `--vehicle-asset /path/to/model.glb`.

Example:

```bash
uv run uncertain-racecar-export-replay \
  --scenario package://scenarios/ks_barcelona_layout_gp_dallara_f317_rl_long.yaml \
  --rollout-json output/ppo_barcelona_exact_seed0_videos/nominal_seed0_exact.json \
  --output-dir output/blender_nominal_bundle

uv run uncertain-racecar-render-blender \
  --bundle-dir output/blender_nominal_bundle \
  --output-path output/blender_nominal_preview.mp4 \
  --engine BLENDER_EEVEE \
  --samples 64 \
  --resolution-x 960 \
  --resolution-y 544 \
  --frame-limit 120 \
  --save-blend-path output/blender_nominal_preview.blend
```

The Blender automation script lives in:

- `uncertain_racecar_gym/assets/blender/render_replay.py`

The bundle should contain:

- `trajectory.json`
- `camera_script.json`
- `scene_manifest.json`
- `track_centerline.csv`
- `scenario.yaml`

If `trajectory.json` includes `planner_debug` entries with:

- `candidate_xy` of shape `N x H x 2`
- `final_xy` of shape `H x 2`

then the Blender renderer will also draw the planner internals directly in the scene:

- thin translucent blue splines for the first 100 sampled candidate rollouts
- a thicker orange spline for the final nominal rollout induced by the chosen control sequence

This is useful for MPPI and Smooth-MPPI videos, where the saved cinematic render should expose the same planning behavior that the PyBullet debug renderer shows during runtime.
